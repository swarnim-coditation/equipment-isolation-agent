Equipment Isolation Agent: Current Flow And Design

This page explains how the current Langflow/Plant360 Equipment Isolation flow works, what each node contributes, and what still needs to improve before it becomes a complete isolation boundary planner.

High-Level Idea

The flow starts with a selected equipment tag, traverses the Unigraph/JanusGraph model around that equipment, identifies nearby isolation devices deterministically, resolves their visual positions on the P&ID image, and emits a payload that the existing Isolation Points UI can render.

1. Select Choose equipment Pick an equipment tag from the selector dropdown, such as P-08. The selector reads Equipment vertices from project 274.
2. Traverse Find equipment boundary Fetch equipment, nozzles/components, direct neighbors, and nearby graph paths from Unigraph.
3. Rank Choose candidate isolation devices Filter graph vertices by valve/blind/flange keywords and rank by proximity/depth and confidence.
4. Visualize Resolve bboxes and output UI payload Match candidates to HILT/P&ID nodes, attach bboxes, and send the strict UI data shape.

Current Data Flow

Selection
Equipment Isolation Selector Config → equipment_tags: [selected tag]

The selector dropdown displays friendly equipment options, but downstream receives only the equipment tag.

Graph Access
Unigraph Config → graph274_traversal → Equipment Boundary Fetcher

This connects to JanusGraph at 44.217.77.13:8182 and queries project graph 274.

Boundary Fetch
Equipment vertex → Nozzles/components → Direct neighbors → Traversal samples by depth

For P-08, important boundary components include N1_P08 and N2_P08.

Candidate Finding
graph vertices → keyword filter → dedupe → rank by depth/confidence

Current keywords include valve, gate_valve, ball_valve, blind, spade, and flange.

BBox Resolution
candidate node_id/name → HILT token match → bbox → UI marker

For the corrected P-08 case, HILT token matching resolved all shown candidates on job 2099.

What We Are Actually Achieving Today

Working behavior: Given equipment on the selected P&ID, we can find graph-confirmed nearby isolation candidates and visualize them on the correct drawing.
  • No LLM is choosing valves. Candidate selection is deterministic from graph data.
  • Equipment selection can now come from a dropdown.
  • For P-08, candidate gate valves near the equipment boundary are found and rendered on the P&ID.
  • The UI receives its required rigid payload: job metadata plus isolation_points.

What This Is Not Yet

Not complete yet: This is not yet a full lockout/isolation boundary plan. It is currently nearest-candidate selection.
  • We do not yet prove all inlet, outlet, vent, drain, utility, and bypass paths are isolated.
  • We do not yet classify candidates as suction isolation, discharge isolation, drain isolation, etc.
  • Changing equipment can require changing the P&ID job as well. VE-21 on the wrong P&ID produced off-image bboxes.
  • The current UI only supports tag_number as the marker label, so missing tags show as null unless the output fills a fallback.

Current Langflow Node Responsibilities

Node Purpose Key Output
Unigraph Config Builds JanusGraph connection info and traversal source. gremlin_server_url, traversal_source, project_id
Equipment Isolation Selector Config Fetches equipment tags for dropdown and outputs selected isolation config. equipment_tags, traversal depth, keyword lists
Equipment Boundary Fetcher Finds selected equipment, attached components, direct neighbors, and traversal samples. equipment_boundaries
Isolation Candidate Finder Filters/ranks isolation-device-like graph vertices. candidates
Isolation BBox Resolver Fetches HILT graph for a P&ID job and attaches image bboxes to candidates. candidates with bbox
Final Isolation UI Output Converts ranked candidates into the strict Isolation Points UI payload. data[].isolation_points[]

Recommended Next Design Step

Move from “nearest valves” to “path boundary isolation”

  • For each equipment nozzle/component, walk outward along physical connections.
  • Stop at the first valid isolation device per path.
  • Group results by source component/path instead of globally ranking all nearby valves.
  • Return roles such as suction isolation, discharge isolation, vent/drain isolation, and utility isolation.

Make equipment selection job-aware

  • Dropdown should show equipment plus drawing/job context.
  • Example: P-08 | pnid_1_bio_final | centrifugal_pump.
  • Selecting equipment should also set or suggest the matching P&ID job id.
  • This prevents selecting VE-21 while still visualizing job 2099.
